# Factory Downtime Root Cause Analysis using Tableau & 7 QC Tools

## Overview
This project focuses on analyzing machine downtime across multiple factories to identify key causes of operational inefficiencies and provide data-driven recommendations. Using Tableau and the 7 Quality Control (QC) tools, the analysis highlights patterns, trends, and root causes impacting production performance.


## Problem Statement
Frequent machine downtime leads to reduced productivity and increased operational costs.  
This project aims to:
- Identify major contributors to downtime  
- Analyze patterns across factories and device types  
- Provide actionable insights to improve efficiency  


## 🛠 Tools & Technologies
- Tableau (Dashboard & Visualization)  
- Data Analysis Techniques  


## Dashboard Features

### KPI Overview
- Total Downtime  
- Average Downtime  
- Top Failure Device  

### Downtime Analysis
- Downtime by Factory  
- Downtime by Device Type  

### Advanced Analytics

#### Pareto Analysis
Identifies top device types contributing to the majority of downtime (80/20 rule).


## Key Insights
- A small number of device types contribute to the majority of downtime (Pareto principle)  
- Significant downtime spikes indicate unstable operational periods  
- Certain machine conditions correlate with increased failure rates  
- Downtime distribution shows presence of extreme failure cases  


## Recommendations
- Prioritize maintenance for high-failure device types  
- Implement preventive maintenance strategies  
- Monitor critical operational parameters (e.g., temperature)  
- Improve process stability through continuous monitoring  


## How to Use
1. Download the `.twbx` file  
2. Open using Tableau Desktop / Tableau Public  
3. Interact with dashboard filters and visualizations  


## Conclusion
This project demonstrates how data visualization and QC tools can be used together to transform raw operational data into actionable insights, enabling better decision-making and improved efficiency.
